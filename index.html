How it works.

A systemd timer, a bash watchdog, and a tmux session — that's the entire runtime.

1

systemd timer

Fires every 60 seconds. Calls the watchdog script. Survives reboots, logouts, and crashes.

claude-channels.timer → claude-channels.service
2

ensure-claude.sh

The watchdog. Checks if the clawbot tmux pane has a live Claude process. If not, recreates everything. If Claude is stuck (spinner frozen for ~2.5 min), kills and restarts.

Level 1: crash detection → Level 2: hang detection
3

claude-channels-loop.sh

Runs inside tmux. A while true loop that restarts Claude Code with channel plugins after any exit. 5-second backoff prevents CPU spinning.

claude --channels plugin:discord plugin:telegram
4

Claude Code CLI

The actual agent. Runs with --dangerously-skip-permissions and a curated deny-list. Responds to messages, executes tasks, maintains memory.

Discord + Telegram → Claude → tools → response

Two-level failure detection.

Most watchdogs check if a process exists. Clawbot checks if it's actually doing anything.

Level 1

Crash detection

Uses ps -s <pane_pid> to scope the process search to the tmux pane's session group. No false positives from other Claude instances. If neither Claude nor the loop script's bash is in that group — kill and recreate.

Level 2

Hang detection

Captures the tmux pane twice, 1 second apart. If both snapshots are byte-identical and the footer shows "esc to interrupt" — the spinner is frozen. Requires 2 consecutive detections (~2.5 min) before killing. No premature restarts.

Memory that persists across sessions.

A file-based memory system organised by type. The agent reads, writes, and updates its own knowledge base — no database required.

user User context

Who you are, your role, preferences, expertise level. Tailors responses to your specific background.

feedback Behaviour corrections

What to avoid, what worked. Includes why, so the agent can judge edge cases rather than blindly follow rules.

project Project state

Ongoing work, goals, deadlines, decisions. Context that isn't derivable from code or git history alone.

reference External pointers

Where to find things in external systems — Linear boards, Grafana dashboards, API credentials, deployment docs.

FAQ
